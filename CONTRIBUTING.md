# Contributing

תודה על הרצון לתרום! המסמך מפרט את הכללים, זרימת העבודה והסטנדרטים הקהילתיים.

## סוגי תרומה
- תבניות UI/UX חדשות או שדרוג תבניות קיימות.
- תיקוני באגים ושיפורי נגישות.
- שיפורי תיעוד והרחבות לנוהלי עבודה.

## זרימת עבודה
1. פתחו Issue לתיאור השינוי או הפיצ'ר.
2. צרו Fork או Branch ייעודי.
3. בצעו שינויים ומבחנים מקומיים.
4. שלחו Pull Request עם תיאור ברור.

## סטנדרט קומיטים
המאגר משתמש ב־Conventional Commits:

```
feat: add new dashboard template
fix: correct spacing in cards
docs: update roadmap milestones
```

## בדיקות
- ודאו שהאתר והשינויים נפתחים ללא שגיאות.
- אם נוספה תבנית, צרפו preview.png ותיאור ב־README פנימי.

## סקירת קוד
- כל PR דורש לפחות מאשר אחד.
- יש לעמוד בדרישות CODEOWNERS וה־CI.

## תקשורת
בשאלות ניתן לפנות ב־Issues או בערוץ התמיכה המתועד ב־SUPPORT.md.
